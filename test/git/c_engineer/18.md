1.开发耗时较长的需求（比如超过一周）需要单独开分支，开发超过一周说明功能已经比较多了，测试和修bug的时间都会比较久，会影响生产；  
2.风险比较大的需求需要单独开分支，比如各产品比较底层的逻辑，牵扯功能比较多的需求；  
3.此类需求上线之前需要提供预警，告知项目进行相应模块的测试；  
麻烦定义开分支判断的详细流程、判断标准以及技术指南，如有必要请安排给开发提供对应培训
